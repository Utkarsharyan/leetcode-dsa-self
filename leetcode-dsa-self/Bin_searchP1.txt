BINARY SEARCH — PATTERN 1
Classic Binary Search (Exact Search + Insert Position + Boundaries)

---

CORE IDEA
Binary search works on a sorted search space.
Every iteration eliminates half of the remaining elements.

Time Complexity: O(log n)
Space Complexity: O(1)

---

1. PATTERN RECOGNITION

Use binary search when:

1. The array is sorted
2. The search space shrinks every step
3. The problem asks for:

   * an element
   * a position
   * first/last occurrence

Common keywords in problems:

sorted array
log n solution expected
find index
find position
first occurrence
last occurrence

---

2. BINARY SEARCH CORE INTUITION

Binary search maintains a search space.

[l .............. r]

Each iteration:

mid = l + (r - l) / 2

Compare nums[mid] with target:

nums[mid] == target → answer found

nums[mid] < target
target must be on RIGHT
l = mid + 1

nums[mid] > target
target must be on LEFT
r = mid - 1

Search space keeps shrinking until:

l > r

---

3. SAFE MID CALCULATION

Never write:

mid = (l + r) / 2

Correct approach:

mid = l + (r - l) / 2

Reason:
Prevents integer overflow.

---

4. CORE TEMPLATE (EXACT SEARCH)

int l = 0;
int r = nums.length - 1;

while (l <= r) {

```
int mid = l + (r - l) / 2;

if (nums[mid] == target)
    return mid;

else if (nums[mid] < target)
    l = mid + 1;

else
    r = mid - 1;
```

}

return -1;

---

5. VARIATION — INSERT POSITION

If the element does not exist, return the position
where it should be inserted.

Example:

nums = [1,3,5,6]
target = 2

Output:

1

Observation:

When loop ends

l > r

The pointer l becomes the correct insertion position.

Template ending:

return l;

---

6. VARIATION — FIRST OCCURRENCE

Used when duplicates exist.

Example:

nums = [5,7,7,8,8,10]
target = 8

Output:

3

Logic:

When target is found, keep searching LEFT.

if(nums[mid] == target){
idx = mid;
r = mid - 1;
}

---

7. VARIATION — LAST OCCURRENCE

Same idea but search RIGHT.

if(nums[mid] == target){
idx = mid;
l = mid + 1;
}

---

8. COMMON MISTAKES

Mistake 1

r = nums.length

Correct

r = nums.length - 1

Mistake 2

Infinite loop with incorrect pointer updates.

Mistake 3

Stopping search immediately when target is found in
boundary problems.

Mistake 4

Accessing nums[mid+1] or nums[mid-1] without boundary checks.

---

9. EDGE CASES

Empty array
Single element
Target smaller than smallest element
Target larger than largest element
All elements same

Example:

nums = [1]
target = 0

Output:

0

12. INTERVIEW THINKING

Whenever you see a sorted array ask:

"Can I eliminate half the search space every step?"

If yes → Binary Search.

---

13. MENTAL MODEL

Think of binary search as shrinking a window.

[ l ............ r ]

Every iteration removes about 50% of the remaining elements.
