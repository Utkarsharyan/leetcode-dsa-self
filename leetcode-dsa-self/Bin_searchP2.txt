BINARY SEARCH — PATTERN 2
ROTATED SORTED ARRAY

---

CORE IDEA

A rotated sorted array is originally a sorted array that has
been rotated at some pivot.

Example

Original:
[0,1,2,4,5,6,7]

Rotated:
[4,5,6,7,0,1,2]

Binary search still works because:

AT LEAST ONE HALF OF THE ARRAY IS ALWAYS SORTED.

This property allows us to decide which half to discard.

Time Complexity: O(log n)
Space Complexity: O(1)

---

1. PATTERN RECOGNITION

Use this pattern when:

1. The array was originally sorted.
2. The array is rotated at some unknown pivot.
3. You need to search/find something in O(log n).

Common keywords:

rotated sorted array
pivot
circular array
search in rotated array

---

2. CORE INTUITION

Every iteration we do two things:

1. Identify which half is sorted
2. Check if the target lies in that half

Search space:

[l ............. r]

mid = l + (r-l)/2

---

3. DETECTING THE SORTED HALF

Condition:

nums[l] <= nums[mid]

This means the LEFT HALF is sorted.

Example:

[4,5,6,7,0,1,2]
l    mid

Sorted part:
[4,5,6,7]

Else the RIGHT HALF is sorted.

Example:

[6,7,0,1,2,4,5]
mid      r

Sorted part:
[1,2,4,5]

---

4. DECIDING WHERE THE TARGET LIES

LEFT HALF SORTED

If

target >= nums[l] AND target < nums[mid]

Then search LEFT

r = mid - 1

Else search RIGHT

l = mid + 1

RIGHT HALF SORTED

If

target > nums[mid] AND target <= nums[r]

Then search RIGHT

l = mid + 1

Else search LEFT

r = mid - 1

---

5. TEMPLATE — SEARCH IN ROTATED ARRAY

int l = 0;
int r = nums.length - 1;

while(l <= r){

```
int mid = l + (r - l) / 2;

if(nums[mid] == target)
    return mid;

if(nums[l] <= nums[mid]){

    if(nums[l] <= target && target < nums[mid])
        r = mid - 1;

    else
        l = mid + 1;
}

else{

    if(nums[mid] < target && target <= nums[r])
        l = mid + 1;

    else
        r = mid - 1;
}
```

}

return -1;

---

6. FIND MINIMUM IN ROTATED ARRAY

Observation:

Compare nums[mid] with nums[r]

Case 1

nums[mid] > nums[r]

Minimum is in RIGHT HALF

l = mid + 1

Case 2

nums[mid] <= nums[r]

Minimum is in LEFT HALF (including mid)

r = mid

Loop condition

while(l < r)

Final answer

nums[l]

---

7. TEMPLATE — FIND MINIMUM

int l = 0;
int r = nums.length - 1;

while(l < r){

```
int mid = l + (r - l) / 2;

if(nums[mid] > nums[r])
    l = mid + 1;

else
    r = mid;
```

}

return nums[l];

---

8. HANDLING DUPLICATES (HARD VARIANT)

Duplicates break the sorted-half detection.

Example

[1,0,1,1,1]

At some point:

nums[l] = nums[mid] = nums[r]

We cannot determine which half is sorted.

Solution:

Shrink search window

l++
r--

Worst case complexity becomes O(n).

---

9. TEMPLATE — ROTATED ARRAY WITH DUPLICATES

while(l <= r){

```
mid = l + (r-l)/2

if(nums[mid] == target)
    return true

if(nums[l] == nums[mid] && nums[mid] == nums[r]){
    l++
    r--
    continue
}

if(nums[l] <= nums[mid]){

    if(nums[l] <= target && target < nums[mid])
        r = mid - 1
    else
        l = mid + 1
}

else{

    if(nums[mid] < target && target <= nums[r])
        l = mid + 1
    else
        r = mid - 1
}
```

}

return false

---

10. COMMON MISTAKES

Mistake 1

Forgetting that one half is always sorted.

Mistake 2

Using wrong comparison:

nums[l] < nums[mid]

Correct:

nums[l] <= nums[mid]

Mistake 3

Not handling duplicates in LC 81.

Mistake 4

Wrong boundary updates like

l = mid
r = mid

Correct updates:

l = mid + 1
r = mid - 1

Mistake 5

Using wrong loop condition in minimum problem.

Correct:

while(l < r)

1. Which half is sorted?

This question alone decides the direction of search.
