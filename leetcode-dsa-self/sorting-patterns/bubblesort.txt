1. CORE INTUITION
============================================================

Bubble Sort works by repeatedly comparing adjacent elements
and swapping them if they are in the wrong order.

After each pass:
→ The largest element "bubbles up" to the end.

Analogy:
Imagine bubbles in water — larger bubbles rise to the top.
Similarly, larger numbers move toward the end.

Key Idea:

* Compare neighbors
* Swap if left > right
* Repeat passes

---

2. STEP-BY-STEP DRY RUN

Example:
[5, 3, 4, 1]

Pass 1:
Compare 5 & 3 → swap → [3,5,4,1]
Compare 5 & 4 → swap → [3,4,5,1]
Compare 5 & 1 → swap → [3,4,1,5]

Pass 2:
Compare 3 & 4 → no swap → [3,4,1,5]
Compare 4 & 1 → swap → [3,1,4,5]

Pass 3:
Compare 3 & 1 → swap → [1,3,4,5]

Sorted.

Important:
After i-th pass → last i elements are sorted.

3. JAVA TEMPLATE (INTERVIEW READY)

class Solution {
public int[] bubbleSort(int[] arr) {
int n = arr.length;

```
    for(int i = 0; i < n - 1; i++) {
        boolean swapped = false;

        for(int j = 0; j < n - i - 1; j++) {
            if(arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }

        if(!swapped) break; // optimization
    }

    return arr;
}
```

}

6. COMMON MISTAKES

❌ Using wrong indices in swap
WRONG:
temp = arr[i]

CORRECT:
temp = arr[j]

---

❌ Forgetting j < n - i - 1
→ Leads to unnecessary comparisons

---

❌ Not using swapped flag
→ Misses best-case optimization

---

❌ Thinking smallest goes first
→ Actually largest settles first

---

❌ Using non-adjacent swaps
→ Breaks bubble sort logic

12. INTERVIEW TALK TRACK

If asked:

"Explain Bubble Sort"

Answer:

Bubble Sort is a simple comparison-based sorting algorithm
that repeatedly compares adjacent elements and swaps them
if they are in the wrong order. After each pass, the largest
unsorted element moves to its correct position at the end.
It has O(n^2) time complexity and is stable and in-place.
